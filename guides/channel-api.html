Cafe24 스마트스토어 쿠팡 오픈마켓·기타 백화점·버티컬 (C)

공통 TIP

  • 가능하면 최고관리자(대표) 계정으로 로그인하세요. 권한이 없으면 메뉴가 안 보입니다.
  • 키·시크릿은 타인에게 메신저로 보내지 말고, Omnify 수집 폼에만 입력하세요.
  • 메뉴 이름이 조금 달라도 “API / 개발자 / OpenAPI / 연동” 키워드를 찾으면 됩니다.

Cafe24 (자사몰) Tier A

수집 폼에 적을 값: Mall ID(관리자 아이디) · Client ID · Client Secret · (가능하면) Access Token

경로 (공식)

developers.cafe24.com → 로그인 → 앱 생성/선택 → 인증(OAuth) → Client ID / Secret Key
  1. Cafe24 Developers에 접속해 쇼핑몰 관리자 계정으로 로그인합니다.
  2. 개발사(자) 등록이 안 되어 있으면 개인/자사 기준으로 등록합니다.
  3. 앱 생성 후 앱을 선택합니다. (이미 Omnify용 앱이 있으면 그것을 선택)
  4. 메뉴 Oauth 인증 → Client ID/Secret Key 확인에서 두 값을 복사합니다.
  5. Mall ID는 보통 어드민 주소의 https://{mall_id}.cafe24.com 앞부분입니다.
  6. Access Token은 OAuth 인증 후 발급됩니다. 어려우면 Client ID/Secret과 Mall ID만 적어도 Omnify가 이어서 안내합니다.

공식: Client ID/Secret 확인

Front API와 Admin API 권한이 다릅니다. 주문·상품·매출 조회에는 보통 Admin API 스코프가 필요합니다.

네이버 스마트스토어 Tier A/B

수집 폼에 적을 값: 애플리케이션 ID · 애플리케이션 시크릿 · (메모에 스토어명)

경로 (공식)

apicenter.commerce.naver.com → 로그인 → 애플리케이션 → 내 스토어 애플리케이션 → 등록/상세
  1. 네이버 커머스API센터에 스마트스토어 계정으로 로그인합니다.
  2. 좌측 애플리케이션 → 내 스토어 애플리케이션으로 이동합니다. (통합 매니저 권한 필요)
  3. 애플리케이션 등록: 이름(예: Omnify), 설명 입력 후 필요한 API 그룹(상품·주문 판매자 등)을 추가하고 등록합니다.
  4. 등록된 앱을 눌러 상세에서 애플리케이션 ID와 시크릿(보기 → 복사)을 확인합니다.
  5. 스토어는 최대 앱 3개까지 등록 가능합니다. Omnify 전용으로 하나 두는 것을 권장합니다.

공식 소개: 커머스API 소개 · 스토어센터: sell.smartstore.naver.com

구버전 ‘오픈API’와 메뉴가 다릅니다. 반드시 커머스API센터의 「내 스토어 애플리케이션」을 사용하세요.

쿠팡 Tier A

수집 폼에 적을 값: 업체코드(Vendor ID) · Access Key · Secret Key

경로 (공식)

wing.coupang.com 로그인 → 우측 상단 판매자ID 클릭 → 판매자정보/추가판매정보 → OPEN API 키 발급
  1. 쿠팡윙(WING)에 로그인합니다. (사업자 인증된 판매자만 발급 가능)
  2. 우측 상단 본인 아이디 클릭 → 판매자 정보 또는 추가판매정보로 이동합니다.
  3. OPEN API 키 발급 → 약관 동의 → 발급을 진행합니다.
  4. 연동업체 / 자체개발 중 하나를 선택합니다. Omnify 연동 시 안내받은 방식(자체개발 IP 또는 지정 연동)으로 진행하세요.
  5. 발급 후 업체코드 · Access Key · Secret Key를 복사해 수집 폼에 붙여넣습니다.

공식: 쿠팡 Open API Key 발급

OpenAPI 키는 판매자ID당 원칙적으로 1세트입니다. 유효기간(예: 180일)이 있으니 만료 전 재발급하세요.

G마켓 · 11번가 · 옥션 등 오픈마켓 Tier B

수집 폼: 판매자 ID · API/앱 키 · 시크릿(있으면) · 메모에 센터 URL
  1. 각 판매자센터에 로그인한 뒤 「API · 연동 · 개발자 · ESM」 메뉴를 찾습니다.
  2. G마켓/옥션(ESM): ESM Plus 등 판매자 포털의 API/제휴 연동 안내를 따릅니다.
  3. 11번가: 셀러오피스에서 Open API 신청·키 발급 메뉴를 사용합니다.
  4. 메뉴를 못 찾으면 폼 메모에 「담당자 연락처 + 판매자센터 URL」만 적어도 Omnify가 대신 확인을 도와드릴 수 있습니다.

SSG · 롯데온 · 올리브영 · 백화점 등 Tier C

직접 API보다 입점 EDI / 사방넷 중개가 흔한 채널입니다.
  1. 자체 공개 API가 약하면, 사방넷에 이미 연동된 채널인지 먼저 확인하세요.
  2. 사방넷 경유 시 → 이 폼의 「물류(사방넷)」+ 사방넷 가이드를 우선 작성합니다.
  3. 직접 입점 API가 있으면 파트너사에서 받은 업체코드·인증키·문서를 캡처/메모로 첨부(폼 메모란)하세요.
티어 D(테무·일부 소셜 등)는 공개 API가 약해 CSV 수동 업로드 방식이 될 수 있습니다. Omnify 담당자와 방식을 합의해 주세요.

가이드 정보는 2026년 기준 공식·벤더 문서를 참고해 작성했습니다. 메뉴 위치가 바뀌면 공식 ‘API 발급’ 검색어로 확인해 주세요.
